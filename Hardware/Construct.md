# 系统整体框架
主要 
* 双声道USB耳放
* 简单的OTA方案，方便散布后进行固件升级

具有Mic功能
能读SD卡
甚至可以利用芯片强大的算力进行DSP操作

## 1. HPM6P41 LQFP-100 作为主控,功能如下:
    1. 利用HPM的usb内核模拟成USB设备，接受并处理音频 
    2. 利用HPM的I2S和I2C与ES9018S DAC芯片进行通讯和配置,由于我们最终只需要双通道，故只需要连接DATA_CLK（位时钟,DATA1(左右声道帧时钟),DATA2(串行数据接口，左右声道都从这里走)
    3. 利用HPM的ADC，对音频信号进行采集，模拟成Mic发回电脑
    4. 利用HPM的PPI或者SPI外设，与8080并口或者SPI串口屏幕进行通讯
    5. 利用HPM的编码器外设和GPIO，读取旋转编码器和按键
    6. 利用HPM的GPIO，控制整个系统的上电顺序
    7. 利用HPM的IIC或者SPI和一颗加速度传感器/重力传感器进行通讯，自动进行屏幕翻转
    8. 利用其QSPI读取SD卡中的音乐
## 注意事项：

    1. 记得给芯片留出放下散热片的空间

## 2. 一代霸主 ES9018 QFN64 作为DAC输出
## 设计要点

    1. VDD_L,VDD_R,左右声道各需要1.2V模拟电源
    2. AVCC_L,AVCC_R，左右声道各需要3.3V的模拟电源 25mA
    3. VDD,1.2V，数字逻辑核心电源 37mA
    4. DVCC_T,DVCC_B，用于顶部和底部的数字 Pad 环供电 ，标称电流为 7mA 。
    5. 上电时序： VDD_L/R和AVCC_L/R --> DVCC --> VDD --> 最后给主时钟MCLK，然后reset抬高至少1ms,才能正确初始化

## 注意事项
    1. 多余的用不到的data脚下拉电阻接地

## 供电方案(推荐用内置DCDC)

1. DC或者PD_12V输入可选
2. 一路DCDC降压到3.3V_SYS SY8368QNC 6A持续，需要打过孔开窗散热
3. 一路DCDC降压到5V_SYS SY8368QNC

* 数字部分
1. 3.3V_SYS 转到1.2~1.25V给HPM6P41内核供电,最高需要1A，感觉热损耗有点高,可能还需要串接二极管进行热量的分担
7. 3.3V_SYS 通过一个LDO转到1.2V给DAC数字供电，需要enable
4. HPM6P41的LDOOTP可以不用，但是电容还是加上吧,LDOPMC并4.7uF
4. MCU 的VIO我真想全接到1.8V，但是条件所限制，全部接到3.3V_SYS吧

* 模拟部分
3. 5V_SYS 通过一个LDO转到3.3V_MCU_VANA给MCU的ADC使用
1. 5V_SYS通过一个LDO加电容倍增器转到3.3V_VCCA，需要enable
6. 3.3V_VCCA通过一个LDO转到1.2V给DAC的模拟部分供电,需要enable
8. 5V通过Buck_Boost拓扑得到-12V，
9. 5V通过Boost升压到+12V

### 电源树

- **输入**: DC 12V 或 PD 12V
  - **DCDC降压** (SY8368QNC, 6A) → **3.3V_SYS**  
    *需打过孔开窗散热*
    - **LDO** (给 HPM6P41 内核) → **1.2~1.25V**  
      *最大需要供电 1A，热损耗较高，可能需串接二极管分担热量*
    - **LDO** (带使能，给 DAC 数字部分) → **1.2V_DAC_DIG**
    - **直接供电**: HPM6P41 的 VIO（全部接 **3.3V_SYS**）
    - *HPM6P41 内部 LDO 相关*:  
      - LDOOTP 不用但需保留电容  
      - LDOPMC 并联 4.7µF 电容
  - **DCDC降压** (SY8368QNC, 6A) → **5V_SYS**
    - **LDO** (给 MCU ADC 模拟供电) → **3.3V_MCU_VANA**
    - **LDO + 电容倍增器** (带使能，低噪声) → **3.3V_VCCA**
      - **LDO** (带使能，给 DAC 模拟部分) → **1.2V_DAC_ANA**
    - **TPS65131Q1：**
        - **Buck-Boost** → **-12V**
        - **Boost** → **+12V**

## 芯片小细节
1. 24Mhz晶振自动使能，没有晶振自动切换内部24MhzRC
1. DCDC_IN脚通过10K电阻接地，禁止浮空。
2. DCDC_LP脚通过10K电阻接地，或浮空。
3. DCDC_GND接地。
4. DCDC_SNS脚通过10K电阻接地或浮空。
5. LDO/DCDC 1.2V 输出接VSOC（注：外部电源供电时需满足芯片数据手册中要求的VSOC电压范围）
6. 选择压差更低的LDO，同时降低LDO 输入和输出电压压差，例如降低输入端电压值或输
入串入二极管或电阻的方式，以分担LDO发热量，必要时请做额外的散热处理。LDO输出电容在电路稳定条件允许
情况下接足够大的电容
7. PDGO是HPM的一个特殊模块，承担了部分时钟管理，软件唤醒，保存一部分掉电数据的功能
